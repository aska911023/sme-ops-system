# 排班系統 — 演算法與規則總覽

> 給 HR / 主管看的完整規則手冊。系統怎麼決定誰上班、誰休、誰加班、薪水怎麼算。

---

## 一、運作模式

### A. 兩種排班週期（看店面設定）
| 模式 | 怎麼算累計 | 單位上限 |
|---|---|---|
| **標準工時** | 每月歸零 | 每週 40h、每月 168h |
| **2 週變形** | 每 2 週歸零 | 每 2 週 84h |
| **4 週變形** | 每 4 週歸零（多數門市） | 每 4 週 160h |
| **8 週變形** | 每 8 週歸零 | 每 8 週 320h |

> 設定位置：排班 → 門市設定 → 變形工時制度（含起算日）。
> 設定變形工時後，系統會以「起算日」為基準每 N 週切一個 cycle，跨 cycle 時自動歸零，**不會把上 cycle 的時數算進新 cycle**。

### B. 兩種派班引擎
| 模式 | 觸發 | 怎麼派 |
|---|---|---|
| **時段覆蓋制** | 門市有設「時段人力需求」 | 系統自己拼班、確保每個時段人力滿足 |
| **班別制** | 門市只有班別定義（早/中/晚） | 從現有班別中挑配 |

## 二、每週派班的 6 大步驟

排班引擎逐週執行，步驟如下：

### 步驟 1：標出「不能上班」的日期
1. 員工**請休**的日子 → 直接休
2. 員工沒設**可上班時段**的星期 → 自動休
3. 該日**最低人力**沒到 → 從上面的休裡面，撤回幾個（兼職優先撤、低 priority 優先撤）讓他們上班
4. **主動分配月休**：依員工剩餘月休配額，挑該日**人力需求最低**的日期給休
5. **全職額外休**：員工數 > 該週最大人力需求 → 全職多給休（兼職不動，因為兼職用個人時數上限控制）

### 步驟 2：派班依據（**評分機制**）
每個員工 + 每個班別組合會算出一個分數，**分數最高的優先派**。

| 因素 | 加分 / 減分 |
|---|---|
| 員工**還沒達月時數目標** | **+15** |
| 員工**已達月目標** | +5 ~ -10 |
| 該班別**還缺人** | **+40 ~ +50** |
| 該班別**已滿** | -3 / 跳過 |
| 員工標記**偏好班別** | **+20** |
| 員工標記**中性班別** | +8 |
| 員工標記**迴避班別** | **直接跳過** |
| 員工**疲勞度高**（>15 點） | -fatigue × 0.3 |
| **假日 / 週末**加班 | -fatigue × 0.5 |
| **連續兩週末**都排到 | **-40** |
| **連續一週末** | -15 |
| 是當日唯一能**開店 / 關店**者 | **+50** |
| **在合理時數區間**（min~max） | **+15** |
| **超過 max** | -20 |

### 步驟 3：彈性補洞
跑完一遍如果還有時段缺人，自動建立彈性班次（短班）補上，不一定按既有班別。

### 步驟 4：跨店借調
員工資料有設「**可支援的其他門市**」 → 該店缺人時系統會嘗試從別店借。借過來的班會標 **跨店** 並記錄原本門市。

### 步驟 5：開店 / 關店人員修正
如果某天的早班沒有「能開店」的人，系統會找休息中能開店的員工跟班次中**不能開店**的人**互換**。關店同理。

### 步驟 6：法律合規驗證
跑完後一次驗證下面所有限制，違規會顯示在班表上方提示，主管可看到並調整。

## 三、法律限制（絕對遵守）

| 編號 | 規則 | 上限 |
|---|---|---|
| L1 | 連續工作天數 | ≤ 6 天 |
| L2 | 兩班之間休息 | ≥ 11 小時 |
| L3 | 單日工時 | 標準 ≤ 8h；變形 ≤ 12h（含加班） |
| L4 | 週工時 | 標準 ≤ 40h |
| L5 | 例假（每週至少 1 天） | 必須 |
| L6 | 月加班總計 | ≤ 46 小時 |
| L7 | Cycle 內總工時 | 按變形種類（84/160/320） |
| L8 | Cycle 內休假天數 | 按變形種類（4/8/16 天） |

## 四、員工時數三層上限（取最嚴）

```
法定上限   ─→  4 週變形 = 160h
店面預設   ─→  排班 → 門市設定 → 月時數上下限（如 175h）
個人上限   ─→  排班 → 排班偏好 → cap [數字] h/cycle（NULL = 不限）
              ↓
       實際使用 = 三者取最低
```

**範例**：4 週變形 + 店面預設 175h + 員工個人 cap 80h → 系統實際以 **80h** 為該員工 cycle 上限。

## 五、假別規則

### A. 是否吃月休配額（公司給的休）

| 假別 | 算月休？ | 為什麼 |
|---|---|---|
| **休 / 例假 / 休息日** | ✅ 算 | 公司給的法定休 |
| **補休** | ✅ 算 | 加班換的休 |
| 特休 | ❌ 不算 | 員工權益 |
| 病假 | ❌ 不算 | 員工權益 |
| 產假 / 陪產假 | ❌ 不算 | 法定假 |
| 婚假 / 喪假 / 公假 | ❌ 不算 | 法定假 |
| 公傷病假 | ❌ 不算 | 法定假 |
| 生理假 | ❌ 不算 | 員工權益 |
| 事假 | ❌ 不算 | 員工自請 |

> **重點**：員工請特休那天，**不影響公司本來該給他的月休配額**。

### B. 薪資計算

| 假別 | 薪資 | 法源 |
|---|---|---|
| 休 / 補休 / 特休 / 婚 / 喪 / 公 / 產 / 陪產 / 公傷 / 會議 | **全薪** 1.0 | 勞基法 §38、§43、§50、§59、性平法 |
| **病假**（普通病假，前 30 天）| **半薪** 0.5 | 勞基法 §43 |
| **生理假** | **半薪** 0.5 | 性平法 |
| **事假 / 無薪假** | **無薪** 0 | 勞基法 §43 |

**薪水公式**：
```
月薪 = 基本月薪
     - 無薪假天數 × 日薪 × 1.0
     - 半薪假天數 × 日薪 × 0.5
     + 加班費（平日 / 例假分段加給）
     - 遲到扣款
     + 全勤獎金（任何請假都歸零）
```

## 七、特殊員工屬性（每個員工可獨立設定）

| 屬性 | 影響 |
|---|---|
| **employment_type** | 全職 / 兼職 — 影響月時數區間、月休配額、加班限制 |
| **schedule_priority** | 排班優先級（1 最高） |
| **can_open** | 是否能開店（早班分配考量） |
| **can_close** | 是否能關店（晚班分配考量） |
| **weekly_target_hours** | 每週**目標**時數（軟性，演算法盡量靠近） |
| **personal_hour_cap** | 每 cycle **絕對上限**（硬性，不能超） |
| **additional_stores** | 可支援哪些其他門市（跨店借調用） |
| **preferred_shifts** | 偏好班別（會優先派） |
| **avoid_shifts** | 迴避班別（不會派） |
| **availability** | 每星期幾的可上班時段範圍 |

## 八、Cycle 跨年 / 跨月特殊狀況

- **同月跨 2 cycle**（例 5 月含 cycle #1 跟 cycle #2）：演算法分別算，不混在一起
- **Cycle 跨月**（例 cycle #2 = 5/29~6/25）：演算法不關心月份分界，按 cycle 算
- **Cycle 跨年**：理論上正常運作（anchor 不變），但 cycleIndex 會持續累加

> 建議實務：把起算日設在**週日或週一**，這樣每週剛好對齊一個 cycle 的 1/4。

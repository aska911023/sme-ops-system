# SME Ops vs 台灣頂級 ERP（文中/鼎新/正航）差距分析與建置路線圖

> 分析日期：2026-07-04
> 基準對象：文中資訊 MERP/NHRS、鼎新 A1 商務應用雲 + Workflow ERP GP、正航一號/正航 ERP
> 本系統定位：F&B 連鎖零售 ERP（React 19 + Supabase，多租戶）
> 系列文件：gap_v1（QA）、gap_v2（CRM/庫存/WMS/POS）、104_vs_OURS（HR）、CRM_ODOO_GAP_PLAN

---

## 一、總體評價

| 面向 | 相對市售頂級品 | 說明 |
|------|--------------|------|
| POS／會員／CRM／LINE | **領先** | 離線 POS、點數/優惠券、分群行銷、LINE 推播、會員 App — 鼎新掌客雲等級以上 |
| WMS／出貨配送 | **領先** | 揀貨/包裝/月台/承運商抽象層，市售 SME 級 ERP 多半沒有 |
| 平台架構 | **領先** | 事件驅動、簽核引擎、稽核軌跡、RBAC、多租戶 — 同級品少見 |
| 進銷存 | **持平** | 單據流完整、四種成本法、批號效期、三單勾稽都有 |
| 財務會計 | **小幅落後** | 總帳/傳票/報表齊全，缺「單據自動拋轉傳票」深度與立沖帳、票據 |
| 台灣稅務法遵 | **落後（急）** | 401/403 有；**電子發票仍是 MIG 3.2 + Turnkey 端點為 TODO 佔位** |
| HR 薪資法遵 | **落後** | 出勤/排班/薪資架構好，缺勞基法加班費引擎、二代健保、特休年資 |
| 製造 | **落後（低優先）** | 有 BOM/製令表但無工序、成本捲算、倒扣領料 — F&B 央廚才需要 |

**一句話結論**：營運面（POS/CRM/WMS）已超越文中/鼎新 SME 級產品；差距集中在「台灣法遵三件套」——電子發票 MIG 4.1、薪資勞基法引擎、票據管理——這些正是文中賴以生存的護城河，也是法律上非做不可的部分。

| 1 | 電子發票 MIG **4.1**（2026-01-01 起 4.0 以下不再收件） | ✅ 全支援 | ⚠️ `src/lib/einvoice.js` 為 **MIG 3.2**、Turnkey 端點 TODO | **關鍵，已逾期** |
| 2 | 折讓單（D0401/D0501）、作廢（C0501/F0501）、字軌配號管理 | ✅ | ❌ 未見 | 高 |

## 三、標配級差距（市售頂級品皆有、非法定但客戶預期）

1. **單據自動拋轉傳票**：銷貨單→發票+應收+庫存+傳票一氣呵成（文中招牌）。本系統各單據與總帳的自動整合深度待補。
2. **立沖帳**：預收/預付、暫收/暫付的立帳與沖銷管理。
3. **進貨費用分攤**（運費/關稅攤入成本，landed cost）— 鼎新列加購模組。
4. **盤點盈虧自動轉傳票**（文中：盤點→盈虧單→傳票）。
5. **條碼/SKU 主檔**：POS 能刷條碼但無 GTIN/條碼主檔管理、出貨條碼檢核。
6. **月加權平均月結成本**：已有 weighted average，但需確認是「月結批次重算」模式（台灣主流），而非僅即時移動平均。
7. **部門/門市別損益表**：cost center 已在傳票明細，缺報表層。

## 四、本系統已領先之處（守住並擴大）

- 離線 POS + 電子發票開立 + 會員/優惠券整合（文中 WebPOS 等級以上）
- LINE 行銷推播/會員通知（`crm-line-send`）、會員 App、分群/滴灌行銷
- WMS 揀/包/出 + 配送調度 + 承運商轉接層（黑貓/新竹/順豐/超商）
- 簽核引擎（代理、條件規則、檢核清單）— 對標鼎新 Workflow 賣點
- 事件驅動架構、稽核軌跡、AI 助理 — 對標鼎新 AiGP 方向

## 五、建置路線圖

### Phase 1 — 法遵補洞（最急，~4–6 週）
> 目標：合法營運無風險。全部是文中/正航的基本盤。

1. **電子發票升級 MIG 4.1**
   - `einvoice.js` XML 產生器升版（F0401/F0501/F0701、D0401/D0501 訊息別）
   - 決策：自建 Turnkey 連線 vs 接加值中心 API（綠界/ezPay/關貿）。**建議先接加值中心**（月費低、憑證/傳輸全代管），Turnkey 留作二期
   - 字軌配號管理（期別、起迄號、餘量警示）、發票作廢/折讓 UI 流程
2. **薪資勞基法引擎**
   - 加班費倍率表（平日/休息日/例假/國定假日，前2h/後2h 分段）
   - 一例一休排班檢核（七休一、單週工時上限）— 掛進既有排班演算法
   - 特休年資自動計算 + 遞延 + 到期折算；補休 balance（FIFO 到期）
3. **二代健保補充保費**：6 類扣費項目、單次給付 2.4 萬門檻、獎金累計超 4 倍投保薪資邏輯，掛入 `payrollCalc.js`
4. **扣繳憑單媒體檔格式驗證**：拿財政部審核軟體實測既有輸出

### Phase 2 — 財會深化（~6–8 週）
> 目標：會計師事務所拿到帳能直接用，追平文中會計模組。

1. **單據拋轉傳票引擎**：定義「單據類型 → 傳票模板」規則表（銷貨、進貨、收付款、盤點盈虧、薪資、折舊），走既有 EventBus，各模組單據確認即產生待過帳傳票
2. **票據管理模組**：應收/應付票據主檔、託收/兌現/退票狀態機、到期日曆、自動傳票
3. **立沖帳**：預收/預付/暫收/暫付科目掛單沖銷
4. **進貨費用分攤**：按金額/數量/重量分攤入庫成本
5. **固定資產對齊稅法耐用年數表**；月結流程加入折舊自動提列
6. **部門/門市損益表** + 月加權平均月結成本重算批次

### Phase 3 — F&B 營運強化（~6–8 週）
> 目標：發揮餐飲連鎖差異化，超越通用型 ERP。

1. **條碼/SKU 主檔**：GTIN、店內碼、秤重碼（2開頭13碼），POS/WMS/盤點全線刷碼
2. **央廚輕製造**：配方 BOM 成本捲算（原料→半成品→成品）、批次生產單、完工倒扣領料（backflush）、實際 vs 標準成本差異
3. **MRP-lite 補貨建議**：門市銷量 + 安全存量 + 供應商前置期 → 自動採購建議單（不做完整 MRP 排程）
4. **食安追溯**：批號正反向追溯報表（原料批號 ↔ 成品 ↔ 銷售單）— F&B 法遵加分

### Phase 4 — 差異化拉開距離（持續）
1. 自訂報表產生器 / BI 下鑽（對標鼎新 BI）
2. 電商訂單整合（蝦皮/momo/官網）→ 既有訂單流
3. 需求預測（銷量 + 天氣/節慶）餵 Phase 3 補貨建議
4. 承運商 API 實接（目前為 adapter 佔位）
5. 開放 API / Webhook，讓事務所系統（文中 WSTP）能拉帳

## 六、優先順序邏輯

```
急迫 × 法定：MIG 4.1 電子發票（已逾期）＞ 勞基法薪資 ＞ 二代健保
高價值 × 差異化：拋轉傳票 ＞ 票據 ＞ 央廚 BOM ＞ 條碼
低優先：完整 MRP、工序排程、模具管理（非 F&B 需求）
```

**不建議做**：分批分步製造成本、託外加工、模具管理、兩岸集團架構 — 那是鼎新 Workflow GP 面向電子製造業的功能，與 F&B 零售無關。

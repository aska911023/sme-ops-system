第 1 章 · 簽核中心 + 加簽功能

不管你是員工、主管還是 HR，每天都會用到簽核中心。這章詳細到「按哪顆按鈕、看到什麼畫面」的程度。

1.1 簽核中心是什麼？

簡單講：把你要簽的單、跟別人要簽你的單，集中在一個畫面看。不用切 8 個頁面找待簽資料。

💡 想像一下： 以前簽請假要進「請假管理」頁、簽費用要進「費用申請」頁、簽離職要進「離職申請」頁... 8 個頁面切來切去。現在通通在儀表板的「我的待簽」一個地方看完。

1.2 第一次使用：從登入到看到待簽

步驟 1 — 用瀏覽器登入

用 admin / HR 給你的帳號密碼登入系統。登入後會看到「儀表板」（也就是首頁）。

步驟 2 — 看儀表板上方的 Tab

儀表板的最上面有兩個分頁（Tab）：

📍 儀表板（首頁）
📊 總覽
📋 我的待簽 3
↑ 上面紅色「3」表示有 3 張單在等你簽
系統會幫你：
  • 有待簽 → 自動切到「我的待簽」（不用自己點）
  • 沒待簽 → 停在「總覽」（看公司資料看板）

步驟 3 — 點「我的待簽」後的畫面

1.5 三顆按鈕詳細說明

✓ 核准按鈕（綠色）

直接點下去 → 推到下一關。沒有確認視窗，按下去就生效。

什麼時候用：單據沒問題、合於規定、可以放行到下一關。

✗ 退回按鈕（紅色）

點下去後畫面會展開一個文字框叫你填退回原因（必填）。

退回原因 *
請填寫退回原因，員工會收到通知並可以修改後重送...
取消 確認退回
退回原因怎麼寫：
  • ✅ 好：「請補附醫療收據照片」（具體可行動）
  • ❌ 差：「資料不齊」（員工不知道缺什麼）
  • ✅ 好：「日期請改為 5/22 之後，5/20 為休息日不計請假」（清楚說明）
  • ❌ 差：「日期錯」（員工不知道哪裡錯）

🪶 加簽按鈕（橘色虛線）

「加簽」意思是暫時不簽，先請第三人也看一下。詳見 1.6。

1.6 加簽功能：什麼時候用？怎麼用？

什麼時候需要加簽

情境該不該加簽
金額較大，想找會計確認✅ 適合加簽會計師
事項涉及法務，不確定能否核准✅ 適合加簽法務
同事的事，怕利益迴避✅ 加簽另一主管
簡單明確、合規定❌ 直接核准就好，不要拖
單據有錯❌ 不該加簽，該退回讓員工修

加簽的完整流程（圖解）

A 主管收到費用申請（金額較高）
⬇ 點「🪶 加簽」
展開選人視窗：選「B 會計師」+ 填原因
⬇ 送出
B 會計師立刻收到 LINE「🪶 加簽請求」卡
⬇ B 在 LINE 卡上直接點「✓ 核准加簽」
A 主管收到 LINE「B 已加簽通過，請繼續您的簽核」
⬇ A 回 modal 點核准
流程繼續到下一關

加簽操作步驟（逐步）

1明細視窗底部點 🪶 加簽
2畫面就地展開選人視窗，不會跳新視窗
3「加簽人」下拉 → 搜尋同事姓名（系統會自動排除你自己 + 申請人）
4「加簽原因」textarea → 寫明為什麼要請他幫看（建議填，方便對方判斷）
5點「送出加簽請求」→ 對方立刻收到 LINE
6你的明細視窗會變成「🪶 加簽中：[對方姓名] · 撤銷加簽」狀態，無法繼續簽

等加簽人簽完會發生什麼

對方動作系統做什麼你會收到什麼
核准加簽chain 繼續，回到你這關LINE「✅ 加簽已通過，請繼續您的簽核」
退回加簽整張單退回（員工要重提）LINE「❌ 加簽人退回了此單」
還沒動作等你可以「撤銷加簽」改回自己簽

撤銷加簽

如果對方拖太久 / 你後來覺得不用加簽：

1明細視窗會顯示「🪶 加簽中：[對方] · 撤銷加簽」
2點「撤銷加簽」→ 確認
3系統會：
  • 清除加簽紀錄
  • 通知對方 LINE「已撤銷加簽，無需處理」
  • 你的明細視窗回到 3 顆按鈕狀態

1.7 你是加簽人的時候

如果你被別人「加簽」邀請，會看到的：

LINE 上會收到

🪶 加簽請求
#42 · 加簽 #7
待你處理
A 主管邀請你協助加簽
案件：費用申請 - 設備採購
申請人：陳大華
金額：NT$ 25,000
📝 加簽原因
金額較高，請會計師確認預算
✓ 核准加簽 📋 查看 / 退回

怎麼處理

  • 核准加簽：直接在 LINE 卡上點「✓ 核准加簽」就好，不用進系統
  • 退回：要填原因，必須點「📋 查看 / 退回」進 LIFF 操作
注意：退回加簽 = 整張單退回。會通知申請人 + 原本邀請你的主管。是大動作，慎用。

1.8 列印簽呈 PDF

每張單的明細視窗右上角有「下載簽呈」按鈕：

1點右上「下載簽呈」
2瀏覽器彈出列印視窗
3選印表機 / 存 PDF

簽呈格式照台灣傳統公文：申請人 → 第 1 關 → 加簽（如有）→ 第 2 關 → ... → 最終核可，全部蓋章 / 日期都印出來。

1.9 常見問題

Q：為什麼我點「核准」沒反應？

可能原因：

  • 這張單有加簽進行中 → 要等加簽人先簽完。畫面會顯示「加簽中：[name]」
  • 這張單已經被別人簽掉了（多人 approver 情境）
  • 網路問題 → 重整試試

Q：為什麼我看不到應該要簽的單？

可能原因：

  • 還沒輪到你（要前面的人先簽完才會輪到）
  • 這張單的「指定簽核人」設定不對，找 HR 檢查簽核流程設定

Q：手機可以簽嗎？

可以！3 種方式：

  1. 直接在 LINE 卡上點「核准」
  2. 進 LIFF（LINE 內建瀏覽器）的簽核中心
  3. 用手機瀏覽器登入系統 Web 版

Q：我加簽找錯人怎麼辦？

立刻點「撤銷加簽」→ 對方會收到撤銷通知，你可重新發起加簽找對的人。

HR Hub → 點對應功能（如「請假」）
⬇
頁面右上「+ 新增」按鈕 → 展開表單
⬇
填欄位（日期 / 原因等）→ 「送出申請」
⬇
下方列表顯示這筆狀態：待審核
⬇ 等主管簽核
通過 → 已核准，LINE 收通知

2.5 每種申請填什麼

請假申請

欄位說明
假別事假 / 病假 / 特休 / 婚假 / 喪假 / 生理假 ...
起 / 訖日期同一天請完整天就兩個一樣
原因簡單寫即可（事假可寫「個人事務」）

2.8 常見問題

Q：LIFF 顯示「找不到員工資料」

表示你的 LINE 還沒綁定員工。輸入「/註冊 你的姓名」或請 HR 後台幫你綁。

Q：打卡顯示「不在範圍內」

確認：

  1. 手機 GPS 開了沒
  2. 有沒有給 LINE「位置權限」
  3. 你真的在公司 / 門市嗎（系統設定的範圍是「距離 100 公尺內」）

Q：申請送出後一直沒人簽

正常情況主管會收到 LINE 通知。如果他關通知、不在現場 — 用 LINE 直接 ping 他、或請 HR 從後台幫忙催。

Q：可以撤回已送出的申請嗎？

只能撤回「待審核」狀態的單。在列表上點「🗑 撤回」即可。已被簽核的單不能撤，需請主管退回後再修改。

3.3 建一條簽核流程

1左側選單 → 專案流程 → 簽核設定
2點「+ 新增簽核流程」
3填基本資料：
  • 名稱：例如「請假簽核 - 一般員工」
  • 類別：請假 / 加班 / 費用申請 / 離職 ...
  • 金額範圍（如果是費用相關）
  • 啟用：勾起來才會生效
4新增「步驟」— 每一關該由誰簽

3.4 每關該由誰簽 — 9 種設法

每一關都可以選 9 種不同的「指定方式」，看你需要什麼：

固定的人 / 角色 / 部門

設法意思範例
固定員工永遠由某一個特定員工簽所有單都由「林經理」簽
固定角色有該角色的任一人都可簽任何 admin 都可簽
固定部門該部門任一人都可簽會計部任一位都可

動態解（依申請人決定）

設法意思範例
申請人的部門主管系統自動找出申請人所屬部門的主管業務部員工申請 → 業務部主管簽
申請人的門市店長申請人所在門市的店長A 店員工申請 → A 店店長簽
申請人的科別主管更細的分層業務一科員工 → 業務一科主管

指定的部門 / 門市 / 科別主管

設法意思範例
指定部門主管由某個指定部門的主管簽（不管申請人在哪部）所有單都由人資部主管簽
指定門市店長由某個指定門市的店長簽所有單都由台北旗艦店店長簽
指定科別主管同上更細由總公司業務一科主管簽
💡 設計建議：
  • 多關不要重複用同一種「動態解」（會解出同一個人 → 多關形同一關）
  • 第 1 關常用「申請人的部門主管」（最直屬）
  • 第 2-3 關用「固定員工」或「指定部門主管」（如人資、財務）

3.5 費用申請的「金額分流」

同類型可以建多條流程，依金額自動套用：

流程名稱金額範圍關數
費用申請 - 小額0 - 5,0001 關（直屬主管）
費用申請 - 中額5,001 - 50,0002 關（主管 + 財務）
費用申請 - 大額50,001 以上3 關（主管 + 財務 + 總經理）

員工提申請時，系統自動依金額挑流程。員工不用選。

3.6 修改既有流程

⚠️ 注意：修改流程只影響之後新提的單。已經在流程中的單會繼續用原流程跑完，不會被改的流程「半路換軌」。

3.7 常見問題

Q：員工提了單但沒人收到通知

  1. 確認流程「啟用」勾了沒
  2. 第 1 關設定的「指定方式」能不能解出人（如「申請人的部門主管」要部門有主管）
  3. 該員工的 LINE 有沒有綁定

Q：應該簽的人說他看不到單

  1. 檢查該員工是否真的符合本關的「指定方式」
  2. 檢查員工資料的「角色」是否正確
  3. 請該員工關掉再重開系統重整（權限是登入時抓的）

✅ 好的退回原因

第 5 章 · 排班系統使用

給 店長 / HR。員工不需要看這章（員工只需要：知道怎麼提希望休、看自己班表）。

5.1 排班的基本概念

月制工時（不是週制）

系統以月為單位算總工時上限，不是每週固定。為什麼？

  • 每週固定不彈性（連續週切點、月底邊界處理麻煩）
  • 月制可以這週多上、下週少上，自由調配

勞動法依據：每月正常工時 174 小時、加班上限 46 小時（一般行業）。

時段覆蓋制（不是員工選班別）

排班思考方式：

❌ 舊思維
員工各自選自己想上的班別
店長依員工選擇排
結果：人力不平均
✅ 新思維（系統採用）
店長先定「店時段」需求
系統把員工分配進時段
確保每個時段都有足夠人手

三級偏好（員工可表達意願）

員工對每個時段 / 班別可標記偏好：

偏好icon意義
希望⭐強烈想上這時段
可以✅普通可接受
不行❌絕對不能排（如照顧家人時段）

系統會盡量配合 ⭐，避開 ❌。多數員工標準是混合 ⭐ 跟 ✅。

5.2 設定排班參數（店長 / HR）

步驟 1 — 設定店時段

1左側選單 → HR → 排班管理 → 排班規則 / 工時設定
2選門市
3「店時段設定」→ 新增時段
  • 時間範圍：例「10:00 - 14:00」
  • 需求人數：例「需要 2 人覆蓋」
4常見配置：早班、午班、晚班、大夜班

步驟 2 — 設定班別（標準工時）

1「班別管理」→ 新增班別
2填：
  • 班名（例「早班 A」「晚班」）
  • 起 / 訖時間（例 09:00 - 17:00）
  • 顏色（給班表上區分用）
  • 適用員工類型（全職 / PT / 全部）

步驟 3 — 設定每月工時 / 月休

1「工時設定」
2填：
  • 每月正常工時上限（例 174 小時）
  • 每月加班上限（例 46 小時）
  • 正職月休天數（例 10 天）
  • PT 月休天數（例 14 天）

步驟 4 — 員工偏好

1方法 A：店長後台代設（員工個別資料 → 偏好）
2方法 B：員工自己在 LIFF 設定
3每個班別 / 時段標 ⭐ / ✅ / ❌

5.7 常見排班問題

Q：跑出來人力不夠

Q：某員工工時破表

  • 檢查班別覆蓋設計
  • 手動拖拉微調

6.2 薪資怎麼算（公式）

月薪正職

實領 =
　 底薪 (base)
+ 職務津貼
+ 其他津貼（伙食 / 交通 / 全勤）
+ 加班費（依加班類別分桶計算）
+ 績效獎金

－ 投保自付（勞保 + 健保）

PT 兼職

正職月薪 30,000時薪 = 30,000 ÷ 30 ÷ 8 = 125 元

6.3 加班費規則（重要 — A / B / C 三類）

台灣勞動法把加班分 3 種類型，倍率不同：

A 類：平日延長工時

就是平日下班後繼續加班。

時段倍率說明
前 2 小時1.34下班後第 1-2 小時
2 小時後1.67第 3 小時起

範例：平日下班加班 3 小時 → 前 2 小時 × 1.34 + 第 3 小時 × 1.67

B 類：休息日加班

休息日通常指週六（公司可調整）。

時段倍率說明
前 2 小時1.34週六加班前 2 小時
2-8 小時1.67第 3-8 小時
8-12 小時2.67第 9-12 小時

範例：週六出勤 10 小時 → 前 2 小時 × 1.34 + 中間 6 小時 × 1.67 + 後 2 小時 × 2.67

C 類：國定假日 / 例假日上班

⚠️ 注意 — 跟加班費不同： 國定假日上班本身就是「工資加倍」（不是用加班費倍率算），即使只上 1 小時也是這天工資全給一倍。
對象怎麼算
月薪正職底薪已含整月（含國定假日）→ 不額外加
PT 兼職工時 ×1 倍時薪 + 額外 ×1 倍 = 共 ×2

範例 1：月薪 30,000 員工 4/4 兒童節上班 4 小時 → 不另外給（底薪已含）

範例 2：PT 時薪 200 在 4/4 兒童節上班 4 小時 → 4 × 200 (工時) + 4 × 200 (加倍) = 1,600

D 類：例假日（一般指週日）

跟 C 類同邏輯 — 工資加倍。一般不應出勤，必要時須事先取得勞工同意。

加班怎麼分類？

系統會依加班日期自動判斷分類：

  • 週一到週五（非國定）→ A 類
  • 週六（休息日）→ B 類
  • 國定假日 / 週日 → C 或 D 類

HR 不用手動分類。

加班費 vs 補休

員工提加班時可選：

  • 領加班費：依上面 A/B/C 倍率計算金額
  • 換補休：時數進「假別餘額」，之後可用來請特休

給 HR。新員工到職要做的全部設定步驟。

7.1 入職前準備

1跟新員工確認職位 / 部門 / 門市 / 上班日 / 薪資結構
2準備好 LINE 官方帳號 QR code（給員工掃）
3準備好系統登入帳號密碼

7.2 系統建檔

步驟 1 — 建員工資料

1左側選單 → 人員組織 → 員工 → 新增
2填基本資料：姓名 / 英文名 / 身份證 / 生日 / 電話 / Email
3關聯：所屬公司 / 部門 / 門市 / 職位
4填薪資結構：底薪 / 津貼 / 健保級距

步驟 2 — 設定角色（重要！）

角色給誰
store_staff（門市員工）第一線店員 / PT
office_staff（辦公室員工）非主管的辦公室人員
manager（主管）店長 / 部門主管
admin（管理員）HR / 系統管理員
super_admin（最高管理）老闆 / 系統擁有者
原則： 選對角色，員工就自動有對的權限。大多數情況不需要個別調整。

7.3 建系統帳號

1左側選單 → 系統設定 → 使用者管理
2新增使用者 → 關聯到剛才建的員工
3設定初始密碼（要員工首次登入後改）
4把帳號密碼安全地給員工

7.4 LINE 綁定

1給員工 LINE 官方帳號 QR code 加好友
2員工加好友後輸入「/註冊 姓名」綁定身份
3HR 在後台確認：員工 → LINE 帳號 → 「已綁定」
4叫員工試提一張請假單測試會不會收到通知

7.5 加入排班

1HR → 排班管理 → 排班 → 選對應月份門市
2把新員工加入該月排班
3分配班別 / 工時
4請員工到 LIFF 設定偏好（⭐ / ✅ / ❌）

7.6 入職第一週 checklist

項目怎麼確認
✅ 能登入系統 Web讓員工試打卡 / 看自己薪資結構
✅ LINE 已綁定HR 後台 → 員工 → LINE 帳號狀態
✅ 能看到自己班表LIFF HR Hub → 我的班表
✅ 能提請假申請請員工提一張測試（隨後取消）
✅ 試用期評核設定HR → 人才發展 → 試用期管理

7.7 離職員工處理

1員工自行提離職申請（LIFF）或 HR 代提
2核准流程跑完 → 系統自動把員工狀態改成「離職」
3HR 補：薪資結算 / 資遣費（如有）/ 法扣（如有）/ 退保
4停用 LINE 綁定 + 使用者帳號（不要刪，保留歷史資料）

第 8 章 · LINE BOT 指令清單

給 所有員工。在 LINE 對話視窗可以做什麼。

8.1 文字指令

指令功能範例
/註冊 姓名綁定 LINE 帳號到員工/註冊 王小明
/我的單查看自己最近的申請狀態—
/我的待簽查看輪到我簽的單—
/我的班表看本月班表（卡片）—
/薪資查最新一期薪資（卡片）—
/打卡跳 LIFF 打卡頁—
/請假跳 LIFF 請假頁—
/幫助列所有可用指令—

8.2 卡片上的按鈕（不用打字）

LINE 收到的卡片上有按鈕，點下去會：

按鈕動作
✅ 核准對該單核准（不用進 LIFF）
❌ 退回跳出常用原因選項，或選「自己寫」進 LIFF
📋 明細跳 LIFF 看完整資料
✏️ 編輯重送（被退回後）跳 LIFF 編輯重送
🪶 核准加簽對加簽請求直接核准

8.3 通知卡片類型

卡片什麼時候收到
簽核請求卡輪到你簽某張單
加簽請求卡有人邀請你協助加簽
已核准通知自己的單通過了
已退回通知自己的單被退回（含原因 + 編輯重送鈕）
到期提醒排程提醒當日待簽（每天早上推）
任務確認卡任務流程輪到你確認

8.4 LINE 卡看不到 / 收不到怎麼辦

狀況檢查
有些卡看不到舊內容LINE 有快取，輸入指令重新觸發（例 /我的待簽）
完全收不到通知檢查：LINE 通知總開關 / 沒封鎖官方帳號 / 沒解除綁定
輸入 /我的單 沒回應表示未綁定，重新 /註冊 姓名
確定有綁定但收不到請 HR / Admin 後台檢查 LINE 帳號狀態

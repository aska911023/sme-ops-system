1. 權限制度速覽

本系統採用 5 角色 + 個別覆寫 制度。每位員工歸屬一個角色，角色帶有預設權限；如果某個員工需要例外，再「個別調整」覆寫角色預設。

角色定位典型權限
super_admin最高層 / 系統擁有者全部功能
admin系統管理員HR / 經費 / 流程全管
manager主管 / 店長HR 簽核 / 排班核准 / 自單位資料
office_staff辦公室員工看多數資訊、改自己的申請
store_staff門市員工員工 portal、看薪資/班表、提申請

2. 怎麼進到權限設定頁

1用 admin 或 super_admin 帳號登入
2左側 sidebar → 系統設定 → 權限設定（或 /system/employee-permissions）
3左側列員工，點要設的人 → 右側顯示該員工所有模組權限

3. 「查詢」「修改」兩顆按鈕

按鈕意思連動規則
✓ 查詢 看得到資料但不能改 點關 → 自動把修改也關（看不到怎麼改）
✓ 修改 看得到 + 可以改 點開 → 自動把查詢也開（改了一定看得到）

4. 狀態色碼說明

標籤意思例子
角色預設沒做個別調整，照角色來絕大多數情況
手動調整（綠）角色沒給，個別加給給某門市員工特權看薪資
手動調整（紅）角色有，個別禁用某主管暫不能改員工資料

有手動調整的 row 會用對應顏色框住，右側 ↻ 點下去就移除調整、回到角色預設。

5. 所有權限項目逐一介紹

下表 危險度 表示影響範圍：

  • 低：只影響該員工看到的資料
  • 中：可改個別資料或核可表單
  • 高：影響全公司資料 / 計算 / 制度

6. 常見操作場景

情境 A：給某員工加額外權限

1左側找到該員工，點選
2找到對應的功能 row（例：HR / 員工管理 / 修改）
3點「修改」按鈕 → 變青色（按鈕亮起 + ✓）
4row 旁邊出現 手動調整 綠標籤，整 row 加綠框

情境 B：收回某員工的個別權限

1找到該員工 → 找到被「個別加給」的功能（綠標籤）
2兩種方式：
  • A：點「修改」按鈕關掉 → 紅標籤手動調整（角色預設有時才行）
  • B：點 row 最右側 ↻ → 整個移除個別調整、回到角色預設

情境 C：全部恢復角色預設（清零）

1找到該員工
2右上角「↻ 全部恢復角色預設」按鈕
3確認後清除該員工所有個別調整
不可逆：建議先記錄目前個別調整有哪些。

情境 D：直接改員工角色

1系統設定 → 員工管理 → 編輯員工資料
2修改 role 欄位（不在權限頁做）
3儲存後，該員工「角色預設」整個換新角色
注意：改角色後個別調整不會自動清除 — 仍會疊加。如想完全乾淨，搭配情境 C 清零。

情境 E：批次設定（多個員工同時開）

1權限設定頁右上角切「批次模式」
2從左側勾選多個員工
3右側用「✓ / ✗ pill button」對某功能一次性開 / 關所有選中員工
提醒：批次只能「全部開」或「全部關」單一動作。要混合（給 A 加、給 B 收回）就分兩次。

7. Q&A

Q1：為什麼有些功能只有「查詢」沒有「修改」？

因為該功能設計上只是唯讀（如全公司薪資查詢、AI 離職預測）— 並沒有對應的修改權限。

Q2：為什麼有些功能只有「修改」沒有「查詢」？

因為該動作不是「看資料」而是「做動作」— 例如「批次計薪」「假單核可」「刪除員工」。這類純粹的執行權限不需要查詢項。

Q3：簽核權限要怎麼設？

簽核權限不在這頁設。系統用簽核鏈（chain）決定誰能簽。要調整：專案流程 → 簽核設定。

Q4：改了權限對方要重新登入嗎？

建議。權限是登入時抓的，現場改不會立即反映 — 對方需重整瀏覽器或重登。

Q5：admin 自己的權限可以被改嗎？

技術上可以但強烈不建議 — admin 把自己改沒了會鎖死。要做務必先找 super_admin 備援。

Q6：個別調整紀錄會留嗎？

會。「手動調整」標籤上會顯示 MM/DD。歷史紀錄在 DB 的 employee_permissions.override_at。

Q7：為什麼某員工的「導航顯示」全是綠標籤？

因為導航顯示是 single toggle 權限（只有 edit 沒 view）。角色預設多半是「關」，所以一旦開了就是「個別加給」。這是正常現象。

Q8：感覺權限太多管不完，怎麼辦？

個別調整不要設太多 — 太多代表「角色設定可能有問題」。如果同樣的調整反覆出現在多人身上，直接改角色預設（找 super_admin / 後端工程師調 role_permissions）。

# HR 戰情儀表板 — 規劃文件

> 目標:把散落在 57 個 HR 頁面的數據,整合成**一個大總覽**。老闆 + HR 都看、8 區都涵蓋、
> 要「3 秒看完健康度,有問題再鑽進去」。本文件只規劃,不實作。

## 設計原則

1. **燈號優先**:每個指標配 🟢正常 / 🟡注意 / 🔴要處理。老闆掃燈號就知道哪裡出事,不用讀數字。
2. **頂部摘要 → 分區明細**:最上面一排核心數字(一眼看完),下面 8 區 panel(展開看)。
3. **可鑽取**:每張卡/數字點下去 → 跳對應頁面處理(例如「待審假 5」→ 假單頁)。這就是 HR 的操作入口。
4. **兩視角共用一頁**:老闆看「燈號 + 趨勢箭頭 + 成本/流動率」,HR 看「紅點數字 + 鑽取待辦」。同一份數據,各取所需。
5. **效能**:全部數字走**一支聚合 RPC**(後端算好一次回),不要前端撈一堆大表前端算(呼應 over-fetch 教訓)。

## 版面(從上到下)

### 🔝 頂部摘要列(6 個核心,3 秒看完)
一排大數字 + 趨勢箭頭 + 燈號:

| 在職人數 | 本月淨變動 | 滾動離職率 | 本月人事成本 | 待簽核 | 風險警示 |
|---|---|---|---|---|---|
| 128 人 ↑2 | +3 / −1 = **+2** | 4.2% 🟡 | NT$ 2.53M ↑3% | **7 件** | 🔴 **5 項** |

→ 點「待簽核」進簽核中心;點「風險」展開區 8。

---

### 區 1 ·  人力總覽
- **部門/門市人數**:橫條圖(各部門幾人)
- **組成比**:正職 / 兼職 / 外籍 圓餅
- **本月異動**:入職 N、離職 N、調職 N
- **試用期中**:N 人(🟡 X 人 7 天內到期要評核 → 點進 ProbationTracker)
- 來源:`employees`、`ProbationTracker`、`Transfer`

### 區 2 ·  出勤健康
- **今日**:出勤 / 遲到 🟡 / 未打卡 🔴(即時)
- **本月**:遲到 Top 5 員工(次數)、缺勤天數
- **待處理**:補卡待審 N 件 🟡、異常打卡 N 筆(AttendanceDiffReport)
- 來源:`Attendance`、`AttendanceDiffReport`、`PunchCorrection`

### 區 3 ·  假務
- **待審假單** N 件(點 → 假單頁)
- **🔴 特休到期風險**:X 人共 Y 天,在 MM/DD 前沒休完會強制結算/失效(最該主動提醒的)
- **留停中** N 人(LeaveOfAbsence)
- **補休餘額**總時數 + 即將過期兌現的(CompTimeBalance)
- 來源:`Leave`、`LeaveBalances`、`CompTimeBalance`、`LeaveOfAbsence`

### 區 4 ·  加班 / 工時
- **本月加班**:總時數 / 總費用(vs 上月)
- **🔴 勞檢預警**:單月加班逼近上限(46h)的員工清單 — 超了會違法
- **加班 Top 5**:誰加最多(看是否人力不足或異常)
- 來源:`Overtime`、`LaborInspection`

### 區 5 ·  薪資成本
- **本月薪資總額**(vs 上月,趨勢)
- **各部門/門市成本佔比**:橫條或圓餅
- **加班費佔比**:加班費 / 總薪資(太高 = 人力配置問題)
- **🟡 投保級距異常**:調薪了但投保沒同步更新的 N 人(InsuranceGradeMonitor)
- 來源:`Payroll`/`salary_records`、`InsuranceGradeMonitor`、`SalaryAdjust`

### 區 6 ·  流動率 / 留任
- **滾動離職率**:近 12 個月趨勢線
- **🔴 離職高風險**:AttritionPrediction 預測的高風險員工 Top N
- **試用通過率**:近 N 個月
- 來源:`AttritionPrediction`、`Resignation`、`employees`

### 區 7 ·  簽核效率
- **待簽分佈**:各類型各幾件(請假/加班/費用/核銷/調職…)
- **平均簽核時長**:從送出到完成(找出流程瓶頸)
- **🔴 卡關單**:停留超過 X 天還沒人簽的(點 → 那張單)
- 來源:簽核中心數據(`get_approval_timeline` 等)

### 區 8 ·  合規 / 到期風險(老闆最該看的一區)
- **🔴 勞檢紅線**:本月有超時 / 例假出勤 / 特休未休未折現的 — 直接是罰款風險
- **🔴 外籍證件到期**:工作證 / 居留證 X 天內到期 N 人(ForeignWorkers)
- **🟡 契約到期**:定期契約工 X 天內到期 N 人(ContractEmployees)
- 來源:`LaborInspection`、`ForeignWorkers`、`ContractEmployees`

## 技術建議(實作時)

1. **一支聚合 RPC**:`get_hr_dashboard(p_org_id, p_month)` 後端一次算完所有區的數字回 JSON,前端一個 round-trip。**不要**每區各自前端撈大表算(會跟費用申請一樣卡 + over-fetch)。
2. **重指標快取**:離職預測、勞檢計算這種重的,可排程(pg_cron)算好存表,儀表板讀快取,不即時算。
3. **權限**:這頁含薪資/成本/個資 → 限 admin/manager(office_staff 視需要),走 RLS + nav 權限。
4. **燈號門檻設定化**:離職率/加班上限/特休到期天數的「紅黃綠」門檻,放設定表別寫死(法規/政策會變)。
5. **鑽取用既有頁面**:每張卡 navigate 到對應 HR 頁(+ ?focus / ?filter),不用重做明細。

## 建議實作分期(雖然你說都要,實作仍可分批上)

- **Phase 1(最有感)**:頂部摘要列 + 區 1 人力總覽 + 區 5 薪資成本 + 區 8 合規風險 → 老闆視角先到位
- **Phase 2**:區 2 出勤 + 區 3 假務 + 區 7 簽核 → HR 操作面
- **Phase 3**:區 4 加班 + 區 6 流動率(離職預測那塊較重,最後上)

每個 Phase 都是「先擴聚合 RPC 回更多欄位 + 前端加 panel」,不互相卡。
